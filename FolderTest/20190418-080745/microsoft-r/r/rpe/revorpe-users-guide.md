          <source>Troubleshooting Debugging Problems</source>
        </trans-unit><trans-unit id="782" translate="yes" xml:space="preserve">
          <source>The visual debugger included with the Revolution R Enterprise RPE is a powerful tool, but there are some circumstances in which it may not provide you the information you’re looking for.</source>

        </trans-unit><trans-unit id="785" translate="yes" xml:space="preserve">
          <source>While stopped at a breakpoint in the visual debugger, the R Console window shows the prompt “Debug&gt; “.</source>
        </trans-unit><trans-unit id="786" translate="yes" xml:space="preserve">
          <source>If you are familiar with R’s browser function, you may be tempted to use its commands at this prompt.</source>
        </trans-unit><trans-unit id="787" translate="yes" xml:space="preserve">
          <source>DO NOT USE browser COMMANDS WHILE STOPPED AT A BREAKPOINT.</source>
        </trans-unit><trans-unit id="788" translate="yes" xml:space="preserve">
          <source>You may get into a state that will require you to shut down the RPE and restart.</source>
        </trans-unit><trans-unit id="789" translate="yes" xml:space="preserve">
          <source>You can, however, type other R expressions at the “Debug&gt; “ prompt, including object names (including local variable names), assignments, and virtually any other valid R expression.</source>
        </trans-unit><trans-unit id="790" translate="yes" xml:space="preserve">
          <source>If you type commands at the “Debug&gt; “  prompt and see no output, check to see if the command you are debugging is redirecting output via sink() or similar functions.</source>
        </trans-unit><trans-unit id="791" translate="yes" xml:space="preserve">
          <source>Re-directed output is incompatible with the visual debugger.</source>
        </trans-unit><trans-unit id="792" translate="yes" xml:space="preserve">
          <source>Another problematic R command is source(); although this command may be used in code being debugged, any script executed with it will always be executed in release mode - so you will not be able to debug into or hit breakpoints in any code defined in the sourced files.</source>
        </trans-unit><trans-unit id="793" translate="yes" xml:space="preserve">

        </trans-unit><trans-unit id="794" translate="yes" xml:space="preserve">
